# 待修复问题清单

## 紧急问题

### 1. 生成按钮状态异常 🔴
**问题**: 给AI发送消息点击生成后出现错误提示，但底下还在生成中状态，无法再次点击重新生成

**症状**:
- API错误后`isGenerating`状态未重置为false
- 用户无法重新点击发送按钮
- 底部一直显示"生成中"状态

**需要修复的位置**:
- `ChatScreen.tsx` - AI回复处理函数
- 所有`try-catch`块中的`finally`都要确保`setIsGenerating(false)`
- 错误提示后应该立即重置状态

**优先级**: 🔴 高 - 影响基本使用

## 朋友圈相关

### 3. 截图描述优化 🟡
**问题**: AI朋友圈分享截图时，图片描述只说"这是一张手机截屏"，没有包含截图中的文字内容

**需要优化**:
- 在生成截图描述时，明确要求AI描述截图中的文字内容
- 参考格式：
  ```
  手机截屏：顶部显示"XXX"标题，
  内容区域显示"XXX"文字，
  底部有"XXX"按钮
  ```

**优先级**: 🟡 中 - 提升真实性

---

### 4. 朋友圈回复提醒 🟡
**问题**: 其他AI回复用户的朋友圈评论时，用户没有收到提醒

**需要实现**:
- 类似微信朋友圈的"有人回复了你"提醒
- 在朋友圈入口显示红点
- 点开后显示哪些评论有新回复

**参考微信**:
```
[朋友圈] 有3条新回复
- AI A 回复了你: "哈哈哈"
- AI B 回复了你: "同意"
```

**优先级**: 🟡 中 - 提升交互体验

---

### 5. 朋友圈评论回复格式错误 🟡
**问题**: 朋友圈评论里AI互相回复的格式不对，有些回复内容以评论形式显示，而不是"某人回复某人"的形式

**当前错误**:
```
AI A: 哈哈哈
AI B: 同意
```

**应该显示**:
```
AI A: 哈哈哈
AI B 回复 AI A: 同意
```

**需要修复的位置**:
- `MomentsScreen.tsx` - 评论渲染逻辑
- `AIMomentsInteractionManager.tsx` - AI回复逻辑
- 检查`replyTo`字段是否正确设置和渲染

**优先级**: 🟡 中 - 显示格式问题
