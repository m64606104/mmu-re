# 角色迁移功能升级说明

## 🎯 问题

之前的角色迁移功能没有导出记忆库内容，也缺少其他重要数据（朋友圈、财务、关系网络等）。

---

## ✅ 解决方案

全面升级角色迁移功能，确保导出角色的**所有相关数据**，实现真正意义上的完整迁移。

---

## 📦 现在包含的完整数据

### 1. **角色基本信息** 🤖
- `id` - 角色ID
- `type` - 对话类型（private/group）
- `name` - 角色名称
- `avatar` - 当前头像
- `lastMessageTime` - 最后消息时间
- `isMuted` - 是否静音
- `groupRemark` - 群备注（群聊）
- `members` - 群成员列表（群聊）

### 2. **角色设置** (characterSettings) ⚙️
- `avatar` - 头像URL
- `originalAvatar` - 原始头像（用于换回）
- `nickname` - 昵称
- `username` - 网名
- `systemPrompt` - 系统提示词
- `personality` - 性格设定
- `languageStyle` - 语言风格
- `languageExample` - 语言示例
- `memoryEvents` - 记忆事件
- **主动发消息配置** (proactiveMessaging)
  - 是否启用
  - 发送间隔
  - 活跃时段
- **记忆系统配置** (memoryConfig)
- **朋友圈记忆配置** (momentsMemoryConfig)
- **上下文配置** (contextConfig)
- **知识库** (knowledgeBase) ⭐️
  - 所有知识库文档
  - 标题、内容、类型
  - 创建和更新时间

### 3. **记忆库** (memoryBank) 🧠 ⭐️ **新增**
完整的AI记忆库数据，包括：
- 所有记忆条目
- 记忆内容和重要性
- 记忆分类
- 关联消息ID
- 自动生成标记
- 总结配置和状态

### 4. **朋友圈数据** (moments) 📱 ⭐️ **新增**
- 所有朋友圈帖子
- 帖子内容、图片描述
- 点赞和评论记录
- 发布时间
- 朋友圈生成设置
- 发布计划

### 5. **AI状态信息** (aiStatus) 🎭 ⭐️ **新增**
- 当前状态（在线/离线/忙碌等）
- 状态文字
- 当前活动描述
- 行为轨迹历史
- 最后更新时间

### 6. **财务数据** (finance) 💰 ⭐️ **新增**
- 当前余额
- 收入和支出总额
- 所有交易记录
- 收入配置（工资、兼职等）
- 财务设置

### 7. **关系网络** (relationships) 👥 ⭐️ **新增**
与此角色相关的所有关系：
- 关系类型（朋友/恋人/家人等）
- 关系描述
- 亲密度
- 互动历史

### 8. **聊天记录** (messages) 💬 **可选**
- 所有消息内容
- 多媒体消息
- 文档消息
- 红包/转账记录
- 订单记录
- 社交平台内容

### 9. **启用的功能** (enabledFeatures) 🔧
- 已启用的所有功能开关

### 10. **统计信息** (stats) 📊 ⭐️ **新增**
- 消息数量
- 记忆数量
- 朋友圈数量
- 知识库文档数量
- 关系网络数量
- 是否有财务数据
- 是否有AI状态信息

## 🔧 技术实现

### 数据收集逻辑

```typescript
// 1. 记忆库
const memoryKey = `memory_bank_${conversation.id}`;
const memories = JSON.parse(localStorage.getItem(memoryKey));

// 2. 朋友圈
const momentsKey = `moments_${conversation.id}`;
const moments = JSON.parse(localStorage.getItem(momentsKey));

// 3. 财务数据
const financeKey = `ai_finance_${conversation.id}`;
const finance = JSON.parse(localStorage.getItem(financeKey));

// 4. 关系网络
const allRelationships = JSON.parse(localStorage.getItem('relationships'));
const characterRelationships = allRelationships.filter(
  rel => rel.personId === conversation.id || rel.targetId === conversation.id
);

// 5. 统计
const stats = {
  messagesCount: conversation.messages.length,
  memoriesCount: memories.length || 0,
  momentsCount: moments?.posts?.length || 0,
  knowledgeBaseCount: conversation.characterSettings?.knowledgeBase?.length || 0,
  relationshipsCount: characterRelationships.length,
  hasFinanceData: !!finance,
  hasAIStatus: !!conversation.aiStatus,
};
```

### 文件命名

- 旧格式: `角色名_角色迁移_2025-11-11.json`
- 新格式: `角色名_完整角色数据_2025-11-11.json`

## 💡 使用建议

### 导出时机

1. **更换设备前**
   - 导出完整角色数据
   - 包含聊天记录

2. **定期备份**
   - 每周导出一次
   - 不包含聊天记录（文件较小）

3. **分享角色**
   - 导出角色核心数据
   - 不包含聊天记录和财务数据

### 文件管理

- **文件大小**
  - 不含消息: 通常 < 100KB
  - 含消息: 根据消息量，可能几MB

- **存储建议**
  - 保存到云盘（iCloud、OneDrive）
  - 保存到本地备份
  - 重要角色保留多个版本
