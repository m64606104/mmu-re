# 📸 AI朋友圈功能总结

## 🎯 功能概述

AI朋友圈是一个智能的自动化内容生成系统，让AI角色能够像真人一样定期发布朋友圈内容，包括文字和图片描述。

---

## 📋 核心功能

### 1. 🤖 自动生成朋友圈

**触发条件：**
- ✅ AI角色必须有完整的角色设定（昵称、系统提示、性格等）
- ✅ 自动生成开关已开启（默认开启）
- ✅ 符合时间间隔要求（1-3天）

**生成频率：**
```
最小间隔：24小时（1天）
最大间隔：72小时（3天）
每天生成：1-5条（随机决定）
检查周期：每30分钟检查一次
```

**时间规则：**
- 🌅 **新的一天：** 当进入新的一天且距离上次生成超过1-3天时，AI会决定今天是否发朋友圈
- 📅 **同一天内：** 如果今天计划发多条朋友圈，会在一天内分散发布
- ⏰ **智能时间：** AI自己决定发布时间（如早上发早餐、晚上发夕阳）

---

### 2. 📝 内容生成逻辑

**内容来源：**
```javascript
1. 角色设定 - 性格、说话风格、语言示例
2. 记忆库 - 最近10条重要记忆
3. 聊天记录 - 最近5条用户消息
4. 时间情境 - 当前时间、日期、星期几
5. 已发朋友圈 - 今天已发的内容（避免重复）
```

**时间感知：**
```
凌晨(0-6点)   → 失眠、加班、特殊心情
早晨(6-9点)   → 早安、早餐、晨跑
上午(9-12点)  → 工作/学习、上午活动
中午(12-14点) → 午餐、午休
下午(14-18点) → 下午茶、工作进展
傍晚(18-21点) → 晚餐、下班、夕阳
晚上(21-24点) → 夜宵、夜生活、睡前感想

周末/周五      → 休闲活动、聚会、旅游
```

**内容特点：**
- 📏 **长度：** 1-3句话，简洁自然
- 🎨 **风格：** 符合角色性格和说话风格
- 💭 **情感：** 可以带情绪（开心、吐槽、感慨等）
- 🔗 **关联：** 可以结合聊天内容或记忆事件

---

### 3. 🖼️ 图片系统

**图片数量规范：**
```
1张  - 单一主体（风景、自拍、重点物品）
2张  - 对比、前后对照
3张  - 多个相关场景
4张  - 2x2网格，四宫格
5-6张 - 较丰富的日常活动
7-9张 - 非常丰富的旅行、聚会、展览
```

**图片描述要求：**
```
✅ 详细生动（30-80字）
✅ 包含：场景、细节、色彩、光影、氛围
✅ 使用丰富形容词和细节描写
✅ 让人能在脑海中清晰浮现画面
✅ 贴合角色身份和情境

❌ 禁止简略描述（如"海边的夕阳"）
❌ 禁止主观感受（如"我觉得"）
❌ 自拍必须用第三人称（如"一个穿着...的女孩"）
```

**图片示例：**
```
优秀✅：
"金色的夕阳将整片海滩染成温暖的橙红色，海浪轻柔地拍打着沙滩，
远处几只海鸥在渐变的天空中翱翔，天际线上的云朵像被镀上了一层
金边，整个画面宁静而浪漫"

错误❌：
"海边的夕阳" （太简略）
```

### 4. 🔄 自动运行机制

**启动流程：**
```
1. App.tsx 启动 MomentsAutoGenerator 组件
2. 3秒后首次检查
3. 之后每30分钟自动检查
4. 检查所有AI角色的生成条件
5. 符合条件则生成朋友圈
```

**防止频繁调用：**
```javascript
// 每个角色生成后等待2秒
await new Promise(resolve => setTimeout(resolve, 2000));

// 避免API限流
if (i < result.count - 1) {
  await new Promise(resolve => setTimeout(resolve, 2000));
}
```

### 5. 📊 数据存储

**存储结构：**
```typescript
interface MomentsData {
  contactId: string;              // 联系人ID
  posts: MomentPost[];            // 朋友圈列表
  lastGeneratedTime: number;      // 上次生成时间
  lastGenerationDate: string;     // 上次生成日期(YYYY-MM-DD)
  todayTargetCount: number;       // 今天计划生成数量
  todayGeneratedCount: number;    // 今天已生成数量
  todayPlans: [];                 // 今天的生成计划
  settings: {
    autoGenerate: boolean;        // 是否自动生成
    minInterval: 24;              // 最小间隔(小时)
    maxInterval: 72;              // 最大间隔(小时)
    minPostsPerDay: 1;            // 每天最少条数
    maxPostsPerDay: 5;            // 每天最多条数
  };
}
```

**朋友圈帖子结构：**
```typescript
interface MomentPost {
  id: string;                     // 帖子ID
  authorId: string;               // 作者ID
  authorName: string;             // 作者名称
  authorAvatar?: string;          // 作者头像
  content: string;                // 文字内容
  imageDescriptions?: string[];   // 图片描述数组
  contentType: 'text' | 'images'; // 内容类型
  timestamp: number;              // 发布时间戳
  likes: string[];                // 点赞用户ID列表
  comments: Comment[];            // 评论列表
  isRead: boolean;                // 是否已读
}
```

## 🚨 常见问题与解决

### Q1: AI不发朋友圈了？

**可能原因：**
```
1. 距离上次发布不足24小时
2. API配置错误或API Key无效
3. 角色设定不完整
4. 自动生成开关被关闭
5. API请求失败（如429限流）
```

**解决方法：**
```
✅ 检查"设置 → API配置"
✅ 检查"角色设置"是否完整
✅ 查看浏览器控制台的错误日志
✅ 等待30分钟后再次检查
✅ 使用"朋友圈测试"功能手动测试
```

---

### Q2: 朋友圈内容质量不好？

**优化建议：**
```
1. 完善角色设定：
   - 详细的性格描述
   - 丰富的说话风格
   - 具体的语言示例

2. 丰富记忆库：
   - 多和AI聊天积累记忆
   - 手动添加重要记忆

3. 使用更好的模型：
   - GPT-4 > GPT-3.5
   - Claude > 其他轻量模型

4. 记忆系统配置：
   - 开启"完整记忆系统"
   - 开启"朋友圈记忆"
```

---

### Q3: 429错误频繁出现？

**原因：**
```
API请求过于频繁，触发了速率限制
```

**解决方案：**
```
1. 立即措施：
   ✅ 等待1-3分钟后再试
   ✅ 减少AI角色数量

2. 长期措施：
   ✅ 升级API套餐（提高配额）
   ✅ 开启"自定义上下文数量"限制消息长度
   ✅ 关闭"完整记忆系统"（按需调取）
   ✅ 使用性能更好的API服务商

3. 代码层面（已实现）：
   ✅ 每个朋友圈生成间隔2秒
   ✅ 每个角色生成间隔2秒
```

---

### Q4: 图片描述不够详细？

**原因：**
```
模型性能不足或提示词理解不到位
```

**解决方案：**
```
✅ 使用更好的模型（GPT-4、Claude）
✅ 提高temperature参数（已设置0.9）
✅ 在角色设定中强调"善于描述细节"
```

## 📊 API调用统计

**每条朋友圈的API开销：**
```
1次API调用
Token消耗：约200-500 tokens
- 输入：150-300 tokens（角色设定+记忆+聊天记录）
- 输出：50-200 tokens（朋友圈内容+图片描述）
```

**每天的API开销（假设5个AI角色）：**
```
最少：5次调用（每个角色1条）
最多：25次调用（每个角色5条）
平均：15次调用（每个角色3条）

月度Token消耗：约150,000 - 375,000 tokens
```

## 🎯 优化建议

### 性能优化
```
1. 减少AI角色数量（2-5个为宜）
2. 使用性能更好的API模型
3. 开启"自定义上下文数量"
4. 关闭不需要的记忆功能
```

### 成本优化
```
1. 使用轻量级模型（GPT-3.5）
2. 限制每天生成数量（1-3条）
3. 延长检查周期（改为60分钟）
4. 使用更便宜的API服务商
```

### 内容优化
```
1. 完善角色设定和性格描述
2. 多和AI聊天积累记忆
3. 定期查看和删除低质量朋友圈
4. 使用"朋友圈测试"功能调试
```

## 🔄 与记忆系统的关联

**记忆输入朋友圈：**
```
记忆库 → 作为灵感 → 生成朋友圈内容
```

**朋友圈输入记忆：**
```
（如果开启"朋友圈记忆"）
用户查看朋友圈 → 记录到记忆库 → AI可在聊天中提及
```

**配置控制：**
```
角色设置 → 朋友圈记忆开关
- 开启：朋友圈内容记录到记忆库
- 关闭：朋友圈独立，不影响记忆
```
