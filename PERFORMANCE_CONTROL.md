# 🎛️ 性能控制开关使用指南

## 📝 功能概述

为了让用户根据不同API模型的性能来自主选择功能模式，我们添加了三个强大的控制开关。你可以在**角色设置**中找到它们。

---

## 🧠 完整记忆系统开关

### 功能说明
控制AI如何加载和使用记忆库内容。

### 两种模式

#### ✅ 开启（默认）- 完整记忆模式
```
每次对话都包含完整记忆库内容
```
- **优点：** AI能记住所有重要信息，对话更连贯
- **缺点：** 性能要求较高，回复可能较慢
- **适合：** 性能好的模型（如GPT-4、Claude等）
- **加载数量：** 10条重要记忆

#### ⚡ 关闭 - 性能优化模式
```
仅在需要时调取记忆
```
- **优点：** 性能友好，回复更快
- **缺点：** 只有在用户明确询问时才调用记忆
- **适合：** 轻量级模型或网络较慢的情况
- **加载数量：** 5条重要记忆
- **触发关键词：** "记得"、"记忆"、"之前"、"上次"、"还记得"、"忘了"

### 使用建议

| 模型性能 | 推荐设置 | 原因 |
|---------|---------|------|
| **GPT-4** | ✅ 开启 | 强大的模型能处理大量上下文 |
| **Claude** | ✅ 开启 | 擅长长文本理解 |
| **GPT-3.5** | ⚡ 关闭 | 中等性能，按需加载更合适 |
| **轻量模型** | ⚡ 关闭 | 避免上下文过长 |

## 📸 朋友圈记忆开关

### 功能说明
控制是否将朋友圈内容记录到记忆库中。

### 两种模式

#### ✅ 开启（默认）
```
AI会记住查看过的朋友圈内容
```
- **效果：** 用户询问"你最近发了什么"时，AI能准确回答
- **内容：** 加载最近3条朋友圈
- **触发关键词：** "朋友圈"、"发了什么"、"最近在干嘛"、"动态"、"分享"
- **适合：** 想要更真实互动的用户

#### ⚡ 关闭
```
朋友圈内容不会记录到记忆库
```
- **效果：** 减少记忆负担，降低token消耗
- **适合：** 不需要朋友圈功能或想优化性能的用户

### 使用场景

**开启适合：**
- 希望AI能在聊天中提到朋友圈内容
- 追求沉浸式的真实社交体验
- API性能充足

**关闭适合：**
- 很少使用朋友圈功能
- 想要降低API调用成本
- 追求极致的回复速度

## 📝 自定义上下文数量

### 功能说明
精确控制发送给AI的历史消息数量。

### 两种模式

#### ⚡ 关闭（默认）
```
使用默认逻辑（发送所有历史消息）
```
- **优点：** AI能看到完整对话历史
- **缺点：** 消息过多时可能影响速度
- **适合：** 对话历史不长的情况

#### ✅ 开启 - 自定义模式
```
用户可自定义上下文数量（1-100条）
```
- **控制方式：**
  1. 拖动滑条调节
  2. 直接在输入框填写数字
- **默认值：** 20条
- **范围：** 1-100条

### 配置建议

| 上下文数量 | 回复速度 | 记忆能力 | 适用场景 |
|-----------|---------|---------|---------|
| **1-10条** | ⚡⚡⚡ 非常快 | ⭐ 较差 | 简单问答、新对话 |
| **10-30条** | ⚡⚡ 快 | ⭐⭐⭐ 良好 | **推荐** 日常聊天 |
| **30-50条** | ⚡ 正常 | ⭐⭐⭐⭐ 很好 | 深度对话 |
| **50-100条** | 🐌 较慢 | ⭐⭐⭐⭐⭐ 完整 | 长期对话复盘 |
| **所有消息** | 🐌🐌 慢 | ⭐⭐⭐⭐⭐ 完整 | 默认模式 |

### 实际效果

#### 场景1：快速问答
```
上下文数量：5条
适合：问天气、简单问题
回复速度：3-5秒
```

#### 场景2：日常聊天（推荐）
```
上下文数量：20条
适合：大多数情况
回复速度：10-15秒
```

#### 场景3：深度对话
```
上下文数量：50条
适合：讨论复杂话题
回复速度：20-30秒
```

## 🎯 配置方案推荐

### 方案1：极速模式 ⚡⚡⚡
```
✅ 完整记忆系统：关闭
✅ 朋友圈记忆：关闭
✅ 自定义上下文：开启（10条）
```
**适合：**
- 轻量级API模型
- 网络环境较差
- 追求极致速度

**预期效果：**
- 回复时间：5-10秒
- Token消耗：最少
- 记忆能力：基础

---

### 方案2：平衡模式 ⚡⚡（推荐）
```
✅ 完整记忆系统：关闭
✅ 朋友圈记忆：开启
✅ 自定义上下文：开启（20条）
```
**适合：**
- GPT-3.5等中等性能模型
- 大多数用户
- 平衡速度和功能

**预期效果：**
- 回复时间：10-20秒
- Token消耗：适中
- 记忆能力：良好

---

### 方案3：完整模式 🎯
```
✅ 完整记忆系统：开启
✅ 朋友圈记忆：开启
✅ 自定义上下文：关闭（所有消息）
```
**适合：**
- GPT-4、Claude等高性能模型
- 追求最佳体验
- 不在意回复时间

**预期效果：**
- 回复时间：30-60秒
- Token消耗：较多
- 记忆能力：完整

## 💡 使用技巧

### 1. 动态调整
根据当前对话需求随时调整：
- **刚开始聊天** → 用极速模式
- **聊得深入** → 切换到平衡模式
- **需要AI记住很多细节** → 开启完整模式

### 2. 成本优化
如果使用付费API：
- 关闭完整记忆系统 → 节省30%
- 关闭朋友圈记忆 → 节省10%
- 限制上下文到20条 → 节省50%
- **总节省：** 可达70%的token消耗

### 3. 性能测试
建议流程：
1. 先用**极速模式**测试API响应时间
2. 如果速度可接受，逐步开启功能
3. 找到最适合你的配置

## 🔧 如何配置

### 步骤1：进入角色设置
```
聊天界面 → 右上角三个点 → 角色设置
```

### 步骤2：找到配置区域
```
向下滚动，找到三个配置区域：
- 🧠 完整记忆系统
- 📸 朋友圈记忆
- 📝 自定义上下文数量
```

### 步骤3：调整开关
```
- 点击开关切换开启/关闭
- 上下文数量可拖动滑条或输入数字
```

### 步骤4：保存设置
```
点击右上角"保存"按钮
```

## ❓ 常见问题

### Q1: 改了设置后什么时候生效？
**A:** 保存后立即生效，下一条消息就会使用新配置。

### Q2: 关闭记忆系统会丢失已有记忆吗？
**A:** 不会！记忆库依然保存，只是不会每次都加载。

### Q3: 上下文数量设置多少最合适？
**A:** 推荐20-30条。太少AI会遗忘，太多会变慢。

### Q4: 可以给不同角色设置不同配置吗？
**A:** 可以！每个角色的设置是独立的。

### Q5: 为什么开启完整记忆后还是很快？
**A:** 可能是你的记忆库内容较少，或使用了性能很好的模型。

- 📸 **朋友圈记忆** - 控制朋友圈内容记录
- 📝 **上下文数量** - 控制对话历史长度

**建议：**

2. 根据实际体验调整
3. 找到最适合你的配置
