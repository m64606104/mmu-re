# 语音识别功能测试指南

## 🧪 测试步骤

### 测试 1: 未启用语音识别（录音+手动输入模式）

**前提条件：** 语音转文字功能未启用（默认状态）

1. 打开聊天页面
2. 点击底部的麦克风图标 🎤
3. 允许浏览器访问麦克风（如果首次使用）
4. 开始说话（会显示录音计时）
5. 点击"停止录音"按钮
6. ✅ **预期结果：**
   - 弹出输入框，标题显示"✍️ 输入语音内容"
   - 提示文字："请输入您想说的内容（未启用语音识别功能）"
   - 文本框为空，需要手动输入
7. 手动输入内容后点击"发送"
8. ✅ **预期结果：** 发送带有语音文件的消息

---

### 测试 2: 配置语音识别

1. 点击右上角设置按钮
2. 滚动到"🎤 语音转文字"部分
3. 打开"启用语音识别"开关
4. ✅ **预期结果：** 显示配置表单
5. 填写测试配置（智谱清言）：
   ```
   API 地址: https://open.bigmodel.cn/api/paas/v4
   API Key: 你的API Key
   识别模型: glm-4-flash
   ```
6. 点击"保存配置"
7. ✅ **预期结果：** 提示"配置已保存"

---

### 测试 3: 启用语音识别（自动识别模式）

**前提条件：** 已按测试2配置语音识别

1. 返回聊天页面
2. 点击麦克风图标 🎤
3. 说话（例如："你好，今天天气真不错"）
4. 点击"停止录音"
5. ✅ **预期结果：**
   - 显示"正在转换为文字，请稍候..."
   - 几秒后弹出识别结果
   - 标题显示"✅ 语音识别成功"
   - 提示："已自动识别，您可以确认或修改内容"
   - 文本框中显示识别的文字
6. 可以编辑文本内容
7. 点击"发送"
8. ✅ **预期结果：** 发送带有语音文件和文字的消息

---

### 测试 4: 识别失败处理

**前提条件：** 语音识别已启用但使用无效的 API Key

1. 进入设置，将 API Key 改为无效值
2. 保存配置
3. 返回聊天，录制一段语音
4. 点击"停止录音"
5. ✅ **预期结果：**
   - 控制台显示错误信息
   - 弹出输入框，标题显示"✍️ 输入语音内容"
   - 提示："自动识别失败，请手动输入您想说的内容"
   - 可以手动输入并发送

---

### 测试 5: 重新录音

1. 录制一段语音
2. 停止录音后，在确认弹窗中点击"重新录音"按钮
3. ✅ **预期结果：**
   - 弹窗关闭
   - 立即开始新的录音
   - 可以重新录制内容

---

### 测试 6: 取消录音

1. 录制一段语音
2. 停止录音后，在确认弹窗中点击"取消"按钮
3. ✅ **预期结果：**
   - 弹窗关闭
   - 不发送任何消息
   - 录音数据被清除

---

### 测试 7: 录音中途取消

1. 点击麦克风开始录音
2. 在录音过程中点击"停止录音"
3. 在弹出的确认框中点击"取消"
4. ✅ **预期结果：** 录音被取消，不发送消息

## 🐛 常见问题

### 1. 麦克风权限被拒绝
**现象：** 点击麦克风后提示"无法访问麦克风"
**解决：** 在浏览器设置中允许网站访问麦克风

### 2. 语音识别一直失败
**现象：** 每次都显示"自动识别失败"
**检查：**
- API Key 是否正确
- API 地址是否正确
- 网络连接是否正常
- API 额度是否用完

### 3. 识别结果不准确
**建议：**
- 在安静环境中录音
- 说话清晰，语速适中
- 避免过长的语音（建议60秒以内）
- 可以在识别后手动修改

### 4. 录音无声音
**检查：**
- 麦克风是否正常工作
- 系统音量设置
- 浏览器是否选择了正确的麦克风设备
