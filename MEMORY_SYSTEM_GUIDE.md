# AI长期记忆系统使用指南

## 📝 功能概述

AI长期记忆系统让AI能够：
- ✅ **记住用户的个人信息**：姓名、年龄、职业、喜好等
- ✅ **记住AI自己说过的话**：AI提到的经历、故事、观点
- ✅ **记住重要的对话内容**：共同话题、约定、有趣的互动
- ✅ **自动总结关键信息**：每25条消息自动提取重要记忆
- ✅ **保持对话连贯性**：避免AI忘记之前聊过的内容或瞎编

## 🚀 功能特性

### 1️⃣ **自动触发总结**

| 触发方式 | 说明 |
|---------|------|
| 消息数量 | 默认每25条消息自动触发一次总结 |
| 可配置 | 可设置为15/25/50/100条消息 |
| 智能提取 | AI自动识别值得记忆的信息 |

**示例：**
```
消息1-24：正常聊天
消息25：触发总结 ⚡
【系统】：AI正在提取记忆...
【系统】：已保存3条新记忆 ✅
消息26-49：继续聊天
消息50：再次触发总结 ⚡
```

### 2️⃣ **记忆分类系统**

系统会将记忆分为以下类别：

#### 关于用户的记忆
| 分类 | 示例 |
|------|------|
| 个人信息 | "用户叫小明，是一名程序员" |
| 喜好 | "用户最喜欢吃火锅，不喜欢吃香菜" |
| 事件 | "用户上周去了迪士尼" |
| 关系 | "用户有个妹妹在上大学" |
| 习惯 | "用户每天晚上11点睡觉" |
| 情感 | "用户最近因为工作压力很大" |

#### 关于AI自己的记忆
| 分类 | 示例 |
|------|------|
| AI经历 | "AI上周在课堂上吐槽老师把PPT放反了" |
| AI观点 | "AI最喜欢的电影是《肖申克的救赎》" |
| AI关系 | "AI的室友是个学霸" |

#### 对话互动记忆
| 分类 | 示例 |
|------|------|
| 对话互动 | "用户和AI约定周末一起去看电影" |
| 共同话题 | "讨论了《三体》这本书" |
| 有趣互动 | "用户开玩笑说要请AI吃饭" |

### 3️⃣ **重要性分级**

| 等级 | 标识 | 说明 | 示例 |
|------|------|------|------|
| HIGH | ⭐⭐⭐ | 非常重要 | 用户的核心信息、重大事件 |
| MEDIUM | ⭐⭐ | 一般重要 | 日常事件、普通对话 |
| LOW | ⭐ | 次要信息 | 可有可无的细节 |

### 4️⃣ **智能记忆管理**

- **最大容量**：每个对话最多保存100条记忆
- **自动清理**：超过容量时优先删除低重要性的旧记忆
- **防重复**：避免记录重复的信息
- **更新机制**：信息有更新时会标注"更新:"

## 🔧 技术实现

### 系统架构

```
用户发送消息
    ↓
AI生成回复
    ↓
检查消息数量是否达到阈值（25条）
    ↓
【是】触发自动总结
    ↓
构建总结提示词（包含最近30条消息+已有记忆）
    ↓
调用AI进行总结
    ↓
解析AI返回的JSON格式记忆
    ↓
按重要性和分类保存到记忆库
    ↓
下次生成回复时，记忆作为上下文加入系统提示
```

### 核心文件

```
src/types.ts                    # 定义MemoryEntry和MemoryBank类型
src/utils/memorySystem.ts        # 记忆系统核心逻辑
src/components/ChatScreen.tsx    # 集成自动总结功能
```

### 关键函数

#### 1. `getMemoryBank(conversationId)`
获取对话的记忆库，如果不存在则创建新的。

#### 2. `shouldTriggerAutoSummary(conversationId, currentMessageCount)`
检查是否应该触发自动总结（是否达到25条消息）。

#### 3. `buildMemorySummaryPrompt(messages, existingMemories)`
构建给AI的总结提示词，包含：
- 最近30条对话
- 已有的记忆（避免重复）
- 详细的提取要求和分类规则

#### 4. `parseMemorySummaryResponse(response)`
解析AI返回的JSON格式记忆数据。

#### 5. `addMemory(conversationId, content, importance, category, autoGenerated)`
添加新记忆到记忆库。

#### 6. `buildMemoryContext(memories)`
构建记忆上下文，用于AI生成回复时参考。

## 🎯 最佳实践

### DO's ✅

1. **启用记忆系统功能**
   - 在对话设置中启用"记忆系统"功能
   - 系统会自动工作，无需手动操作

2. **信任自动总结**
   - AI会自动提取重要信息
   - 每25条消息触发一次
   - 不重要的闲聊不会被记录

3. **自然对话**
   - 像和真人聊天一样
   - 提到重要信息时AI会记住
   - 提到之前的话题时AI能回忆

4. **利用记忆提升体验**
   - AI会记住你的喜好
   - 不会重复问已知的问题
   - 能延续之前的话题

### DON'Ts ❌

1. **不要期待记住所有细节**
   - 系统优先记录重要信息
   - 不重要的细节会被自动过滤
   - 最多保存100条记忆

2. **不要手动管理记忆**
   - 系统全自动运行
   - 无需手动添加或删除
   - AI会智能判断重要性

3. **不要在短期对话中测试**
   - 需要达到25条消息才触发
   - 短对话不会激活记忆系统
   - 建议长期使用以体验效果

## 🐛 常见问题

### Q1: 为什么记忆没有生效？

**可能原因：**
1. 没有启用"记忆系统"功能
2. 消息数量还没达到25条
3. 对话内容没有值得记忆的信息

**解决方案：**
- 确保在功能设置中启用记忆系统
- 多聊一些，达到25条消息后会自动触发
- 提到重要信息（如个人信息、喜好、事件等）

### Q2: AI为什么忘记了之前的话题？

**可能原因：**
1. 记忆库已满（100条），旧记忆被删除
2. 该信息被AI判定为不重要（LOW级别）
3. 记忆总结时AI没有提取该信息

**解决方案：**
- 重要信息多提几次，增加重要性权重
- 如果是AI说的经历，在后续对话中引用它

### Q3: 如何查看当前的记忆库？

**方法：**
打开浏览器控制台，输入：
```javascript
JSON.parse(localStorage.getItem('chat_memory_banks'))
```

可以看到所有对话的记忆库数据。

### Q4: 记忆系统会影响性能吗？

**不会。**
- 总结只在达到阈值时触发（每25条消息）
- 总结是异步执行，不阻塞对话
- 记忆上下文只取前20条最重要的
- 对API调用影响极小
