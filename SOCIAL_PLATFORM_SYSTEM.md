# 完整社交平台内容生成系统

## 🎉 系统概述

全新的**完整社交平台界面生成系统**，不再是简单的链接卡片，而是**真实模拟小红书、知乎、微博APP**的完整界面，包括：
- 瀑布流布局
- 多NPC视角的帖子/回答
- 完整的评论互动
- 弹窗详情展示
- 真实的UI设计

---

## 📱 支持的平台

### **1. 小红书（XiaohongshuFeed）**
- 📕 **双列瀑布流布局**
- **帖子卡片**：封面图 + 标题 + 作者 + 点赞数
- **弹窗详情**：大图 + 完整正文 + 话题标签 + 评论列表
- **底部互动栏**：评论输入 + 点赞 + 收藏 + 评论数
- **真实感**：多NPC作者 + 真实数据 + 生活化内容

### **2. 知乎（ZhihuFeed）**
- 💡 **问答列表界面**
- **回答卡片**：问题标题 + 作者信息 + 回答预览 + 数据
- **详情弹窗**：完整回答 + 评论区 + 互动按钮
- **专业性**：作者简介 + 结构化回答 + 深度评论
- **蓝色主题**：经典知乎配色

### **3. 微博（WeiboFeed）**
- 🎭 **热搜列表界面**
- **微博卡片**：作者信息 + 话题 + 正文 + 图片 + 数据
- **详情弹窗**：完整微博 + 九宫格图片 + 热门评论
- **橙红主题**：微博品牌色
- **热点话题**：# 话题标签 + 认证标识

## 🎭 生成规则

### **1. 多NPC视角（必须）**
- **小红书**：3-6个不同博主
- **知乎**：2-4个不同回答者
- **微博**：3-5条不同用户的微博

### **2. 每个帖子/回答都有评论（必须）**
- 至少2-4条评论
- 评论要有互动性（提问、夸赞、讨论）
- 评论者也是独立NPC
- 评论点赞数合理

### **3. 内容真实感**
- 标题吸引眼球（emoji + 关键词）
- 正文详细充实（80-300字）
- 话题标签相关
- 图片描述具体
- 数据真实（点赞 > 评论 > 转发/收藏）

### **4. 格式严格**
- 必须严格按照指定格式
- 分隔符是竖线 `|`
- 不要漏掉任何字段
- ID要唯一（post1, post2, ans1, wb1...）

