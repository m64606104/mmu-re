# 标点符号分割修复方案

## 🐛 问题描述

用户反馈：中文双引号`"`总是被分到下一个气泡，导致出现如下情况：

```
气泡1：意了一下，声音依旧平铺直叙，不带任何温度。
气泡2："坐。
气泡3："
```

最后一个气泡只有一个闭引号`"`，这是因为消息分割逻辑没有考虑引号配对的问题。

---

## ✅ 解决方案

### 核心思路

在消息分割后，添加**智能标点符号检查和修复**步骤：

1. **检测单独的引号或标点**：如果某个气泡只有引号或标点符号，自动合并到上一个气泡
2. **检测未闭合的引号**：如果某个气泡有开引号但没有闭引号，检查下一个气泡
3. **自动合并配对引号**：将被分割的引号对合并到同一个气泡
4. **自动补齐缺失引号**：如果找不到配对引号，自动补齐闭引号

### 支持的引号类型

- `"` 和 `"` - 中文双引号
- `「` 和 `」` - 日文直角引号
- `『` 和 `』` - 日文空心引号
- `【` 和 `】` - 方括号

## 🔧 技术实现

### 修改文件
`src/utils/messageFormatter.ts`

### 新增逻辑

在`splitMessages`函数中，添加第五步：

```typescript
// 🧠 第五步：智能标点符号检查和修复
const smartFixedMessages: string[] = [];
for (let i = 0; i < restoredMessages.length; i++) {
  const msg = restoredMessages[i].trim();
  if (!msg) continue;
  
  // 1. 检查是否只有引号或标点符号
  const isOnlyPunctuation = /^[""「」『』【】。！？!?.,，；;]+$/.test(msg);
  
  if (isOnlyPunctuation) {
    // 合并到上一个消息
    if (smartFixedMessages.length > 0) {
      smartFixedMessages[smartFixedMessages.length - 1] += msg;
    }
    continue;
  }
  
  // 2. 检查是否缺少闭合引号
  const openQuotes = (msg.match(/["「『【]/g) || []).length;
  const closeQuotes = (msg.match(/["」』】]/g) || []).length;
  
  if (openQuotes > closeQuotes) {
    // 有未闭合的引号，检查下一个消息
    if (i + 1 < restoredMessages.length) {
      const nextMsg = restoredMessages[i + 1].trim();
      if (/^["」』】]/.test(nextMsg)) {
        // 下一个消息以闭引号开头，合并
        smartFixedMessages.push(msg + nextMsg);
        i++; // 跳过下一个
        continue;
      }
    }
    
    // 3. 如果没有找到闭合引号，自动补齐
    const lastOpenQuote = msg.match(/["「『【]/g)?.slice(-1)[0];
    const closeQuoteMap = {
      '"': '"',
      '「': '」',
      '『': '』',
      '【': '】'
    };
    if (lastOpenQuote && closeQuoteMap[lastOpenQuote]) {
      smartFixedMessages.push(msg + closeQuoteMap[lastOpenQuote]);
    } else {
      smartFixedMessages.push(msg);
    }
  } else {
    smartFixedMessages.push(msg);
  }
}
```

## 🧪 测试案例

### 案例1：单独的闭引号
**输入**：
```
意了一下，声音依旧平铺直叙，不带任何温度。\n\n"坐。\n\n"
```

**期望输出**：
```
气泡1：意了一下，声音依旧平铺直叙，不带任何温度。
气泡2："坐。"
```

### 案例2：多重引号
**输入**：
```
他说："你好。\n\n我说："也好。
```

**期望输出**：
```
气泡1：他说："你好。"
气泡2：我说："也好。"
```

### 案例3：引号中的对话
**输入**：
```
她问："你在吗？\n\n"\n\n我没有回答。
```

**期望输出**：
```
气泡1：她问："你在吗？"
气泡2：我没有回答。
```

### 案例4：未闭合的引号（自动补齐）
**输入**：
```
他说："我要走了。\n\n然后就离开了。
```

**期望输出**：
```
气泡1：他说："我要走了。"
气泡2：然后就离开了。
```

## 💡 设计优势

### 1. **智能配对**
- 自动识别开引号和闭引号的配对关系
- 支持多种引号类型（中文、日文、方括号）
- 跨气泡检测和合并

### 2. **自动补齐**
- 如果引号对被分割且找不到配对，自动补齐闭引号
- 确保每个气泡的引号都是闭合的
- 符合正确的书面用语习惯

### 3. **向后兼容**
- 只影响有问题的气泡，不影响正常气泡
- 保持原有的分割逻辑
- 不会破坏已经正确的消息

### 4. **性能优化**
- 只在恢复占位符后进行检查
- 使用简单的正则表达式和字符串操作
- 时间复杂度 O(n)，n为气泡数量

## 🚀 未来扩展

### 可能的改进方向

1. **更智能的上下文感知**
   - 分析引号内容的语义完整性

2. **支持更多标点符号**
   - 括号配对：`(` 和 `)`、`（` 和 `）`
   - 书名号配对：`《` 和 `》`
   - 省略号处理：避免 `...` 被分割

3. **AI学习优化**
   - 在System Prompt中增加引号使用规范
   - 让AI生成消息时就避免引号分割问题

4. **用户自定义规则**
   - 允许用户配置是否自动补齐引号
   - 配置引号合并的行为

## 📝 注意事项

### 已知限制

1. **多层嵌套引号**：目前只处理一层引号，多层嵌套可能不完美
2. **混合引号类型**：如果开引号是`"`但闭引号是`」`，无法自动配对
3. **引号作为内容**：如果引号本身是消息内容（如讨论引号的用法），可能被错误合并

### 建议

- 在AI的System Prompt中添加引号使用规范
- 提示AI避免在分段位置使用引号
- 对于长篇文档，考虑不分割或使用文档格式
