# 智能行为轨迹系统使用指南

## 📋 功能概述

智能行为轨迹系统通过AI自动为角色生成真实、合理的行为轨迹，不再使用硬编码的预设活动。

### ✨ 核心特性

1. **AI状态更新支持详细描述**
   - AI可以在对话中主动更新自己的状态，并带上详细活动描述
   - 自动提取消息中的地点和活动信息

2. **智能行为轨迹生成**
   - 根据角色设定、当前时间、星期几智能生成
   - 自动识别角色类型（学生、上班族、自由职业者）
   - 每90分钟自动检查并更新

---

## 🎯 功能1：AI状态更新支持详细描述

### 使用场景

当AI在对话中提到要修改状态时，系统会自动：
- 识别目标状态（在线、忙碌、休息中、离开、离线）
- 提取活动描述和地点
- 更新状态和行为轨迹

### 对话示例

**用户：** "你怎么住宿舍啊？"

**AI：** "宿舍？大哥你想啥呢，我们公司哪有这福利啊。我要是住在公司宿舍，那我可就成24小时待命了。我这是自己在外面租的房子啦。我这是笔误，对了我把状态改回来。"

**系统自动处理：**
```
✅ AI自主更新状态：在线
行为轨迹：03:00 在线
```

**如果AI说得更详细：**

**AI：** "对了我把状态改成在线，我现在在家休息呢。"

**系统自动处理：**
```
✅ AI自主更新状态：在线 - 在家休息
行为轨迹：03:00 在线 在家休息 📍家
```

### 触发关键词

系统会检测以下模式：
- "状态" + ("改" / "换" / "设置")
- 目标状态关键词：
  - "在线"
  - "忙碌" / "在忙"
  - "休息"
  - "离开"
  - "离线"

### 活动描述提取

系统会自动匹配以下模式：
```typescript
1. "在XX做YY" → "在XX做YY"
   例：在家休息、在公司工作、在图书馆学习

2. 单独活动 → 活动名称
   例：睡觉、工作、学习、吃饭、开会、上课

3. "刚/准备/正在/要做XX" → 活动描述
   例：准备睡觉、正在吃饭、要出门
```

## 🤖 功能2：智能行为轨迹生成

### 系统架构

```
SmartActivityScheduler (后台调度器)
    ↓
每90分钟检查一次
    ↓
判断距离上次更新是否>1小时
    ↓
调用 generateSmartActivity()
    ↓
构建提示词 → 调用AI API → 解析结果 → 更新状态
```

系统自动从角色设定中提取身份：

| 自由职业、自由工作者 | 自由职业者 | 咖啡厅工作、客户会议 |

### API提示词格式

系统会构建如下格式的提示词：

```
你是张小白，一个学生。

【角色信息】
背景：大学生，喜欢打游戏
性格：活泼开朗

【当前时间】
周一 12:30

【任务】
请根据当前时间和你的角色身份，生成一条真实、合理的行为轨迹。

**要求：**
1. 必须符合你的身份和作息时间
2. 必须真实、具体、有生活气息
3. 只生成一条活动，不要多条
4. 格式：[状态]活动描述|地点（可选）

**状态选项：**
- 在线：日常活动、休息、有空
- 忙碌：工作、学习、开会、上课
- 休息中：睡觉、午休、休息
- 离开：外出、路上

**示例（学生）：**
- 早上7:30 → [忙碌]准备去上课|宿舍
- 上午10:00 → [忙碌]正在上高数课|教室
- 中午12:00 → [在线]食堂排队打饭|食堂
...

**现在生成周一 12:30的行为轨迹：**
格式：[状态]活动描述|地点
只输出一条，不要解释。
```

### AI返回格式

AI会返回如下格式：
```
[在线]食堂排队打饭|食堂
```

系统解析后会生成：
```
状态：在线
活动：食堂排队打饭
地点：食堂
```

## ⚙️ 配置说明

### 调度间隔

在 `App.tsx` 中配置：

```tsx
<SmartActivityScheduler
  conversations={conversations}
  apiConfig={apiConfig}
  intervalMinutes={90}  // 每90分钟检查一次
/>
```

建议值：
- `60` - 每小时（频繁）
- `90` - 每1.5小时（推荐）
- `120` - 每2小时（较少）

### 更新触发条件

在 `smartActivityGenerator.ts` 中：

```typescript
// 超过1小时没更新，应该生成新活动
const shouldGenerate = timeDiff > 60 * 60 * 1000;
```

可修改为：
- `30 * 60 * 1000` - 30分钟
- `60 * 60 * 1000` - 1小时（推荐）
- `120 * 60 * 1000` - 2小时

#### 1. `analyzeAndUpdateStatusFromAI`
```typescript

  message: string
```


**流程：**
1. 检查API配置
2. 构建提示词
3. 调用AI API
4. 解析返回结果
5. 更新状态和轨迹

#### 3. `startActivityScheduler`
```typescript
// 启动定时调度器
export const startActivityScheduler = (
  conversations: Conversation[],
  apiConfig: ApiConfig,
  intervalMinutes: number
): NodeJS.Timeout
```

**特点：**
- 定时检查所有角色
- 判断是否需要更新
- 随机延迟避免同时更新

## 🎮 使用建议

### 1. 角色设定要详细

在角色设定中明确身份：
```
✅ 好的设定：
"你是一名大学生，计算机专业大二，平时喜欢打游戏、看动漫"

❌ 不够详细：
"你是个学生"
```

### 2. 让AI自然表达

不需要刻意让AI说"改状态"，自然对话即可：
```
AI："对了我在家呢，刚才写错了"
→ 系统会自动识别并更新
```

### 3. 观察生成效果

查看行为轨迹弹窗：
- 时间是否合理？
- 活动是否符合角色？
- 地点是否准确？

### 4. 调整调度频率

根据使用场景调整：
- **测试阶段**：60分钟（快速看效果）
- **日常使用**：90分钟（推荐）
- **轻度使用**：120分钟（省API调用）

## 🐛 故障排查

### 问题1：不生成行为轨迹

**检查项：**
1. API配置是否完整？
2. 角色是否有characterSettings？
3. 控制台是否有错误？

**解决方法：**
```javascript
// 打开控制台，查看日志
console.log('🤖 启动智能行为轨迹调度器...')
console.log('⏰ 距离上次更新已X分钟，应该生成新活动')
```

### 问题2：生成的活动不合理

**可能原因：**
- 角色设定不够详细
- 提示词需要优化

**解决方法：**
1. 完善角色的systemPrompt
2. 在`smartActivityGenerator.ts`中调整提示词

### 问题3：频繁调用API

**解决方法：**
```tsx
// 增加检查间隔
<SmartActivityScheduler
  intervalMinutes={120}  // 改为2小时
/>

// 或增加更新触发阈值
const shouldGenerate = timeDiff > 120 * 60 * 1000; // 2小时
```

## 📊 性能影响

### API调用频率

假设有3个AI角色：
- 每90分钟检查一次
- 每次检查3个角色
- 平均每个角色1小时更新一次

**估算：**
- 每小时API调用：~3次
- 每天API调用：~72次
- Token消耗：~200 tokens/次 = 14,400 tokens/天

### 优化建议

1. **按活跃度调整**
   - 常用角色：60分钟
   - 偶尔聊天：90分钟
   - 很少互动：120分钟

2. **夜间暂停**
   ```typescript
   const hour = new Date().getHours();
   if (hour >= 2 && hour < 7) {
     console.log('深夜时段，跳过更新');
     return false;
   }
   ```

3. **Token优化**
   - 缩短提示词
   - 减少示例数量
   - 降低max_tokens
