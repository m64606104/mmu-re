# AI 时间感知系统使用指南

## 📝 功能概述

时间感知系统是一个增强AI对话自然性的核心功能，让AI能够：
- ✅ 知道当前的时间和日期
- ✅ 感知距离上一条用户消息过去了多久
- ✅ 分析用户说要做什么事情（如吃饭、睡觉）
- ✅ 根据时间间隔智能选择是否延续之前的话题
- ✅ 模拟真实人类的对话习惯

---

## 🎯 核心价值

### 问题场景

**没有时间感知时的AI：**
```
用户（晚上7点）："我要去吃饭了"
...（过了2小时）...
用户（晚上9点）："回来了"
AI："好的，快去吃饭吧！趁热吃哦~" ❌  // 完全不对，用户已经吃完了
```

**有时间感知的AI：**
```
用户（晚上7点）："我要去吃饭了"
...（过了2小时）...
用户（晚上9点）："回来了"
AI："吃完啦？吃的什么呀~" ✅  // 自然，知道时间已经过去
```

## 🚀 功能特性

### 1️⃣ **行为时长分析**

系统预设了常见行为的时长：

| 行为 | 最短时长 | 最长时长 | 示例 |
|------|---------|---------|------|
| 吃饭 | 20分钟 | 60分钟 | "我要去吃饭了" |
| 睡觉/休息 | 30分钟 | 8小时 | "我去睡觉了" |
| 洗澡 | 15分钟 | 40分钟 | "我去洗个澡" |
| 开会 | 30分钟 | 2小时 | "我有个会议" |
| 上课 | 40分钟 | 2小时 | "我要去上课了" |
| 外出 | 1小时 | 4小时 | "我出门一趟" |
| 运动 | 30分钟 | 2小时 | "我去健身房" |
| 上班/工作 | 4小时 | 8小时 | "我要去上班了" |
| 看电影 | 2小时 | 3小时 | "我去看电影" |
| 打游戏 | 30分钟 | 3小时 | "我打把游戏" |

**智能判断：**
- 如果用户说"我要去吃饭"，过了15分钟就点击生成
  - AI知道："用户应该还在吃饭中或刚刚结束"
  - 回复建议："可以自然询问，不要太急切"

- 如果过了2小时才点击生成
  - AI知道："用户早就吃完了"
  - 回复建议："可以问吃的怎么样，但要轻松随意"

### 2️⃣ **时间间隔分级处理**

| 间隔 | 建议 | 示例场景 |
|------|------|---------|
| < 5分钟 | 直接延续话题 | 实时聊天 |
| 5-30分钟 | 自然延续话题 | 短暂离开后 |
| 30分钟-2小时 | 可延续可不延续 | 中等间隔 |
| 2-6小时 | 不要刻意延续 | 几个小时后 |
| 6-24小时 | 优先新话题 | 半天到一天 |
| 1-2天 | 完全不提具体话题 | 隔天聊天 |
| >2天 | 像久别重逢一样 | 好几天没联系 |

### 3️⃣ **时间段识别**

系统精确识别时间段并给出建议：

| 时间段 | 建议 |
|--------|------|
| 0-5点（深夜） | 正常人应该在睡觉，如果还在聊可能失眠/加班 |
| 5-7点（清晨） | 大部分人刚醒或还在睡 |
| 7-9点（早上） | 适合问候"早上好"，但不要每次都问 |
| 9-12点（上午） | 工作/学习时间，对话可能需要简短 |
| 12-14点（中午） | 午餐休息时间 |
| 14-18点（下午） | 正常工作/学习时段 |
| 18-21点（傍晚） | 下班/放学时间 |
| 21-24点（晚上） | 一天快结束了 |

### 4️⃣ **跨时间点检测**

系统能检测是否跨越了重要时间点：

- **跨越睡眠时间**（晚上→早上）
  - 可以问候"早上好"、"睡得好吗"
  - 但也可以直接聊别的，不是必须的

- **跨越午餐时间**（上午→中午）
  - 可以提及"吃饭了吗"
  - 但真人不会每次都问

- **跨越晚餐时间**（下午→傍晚）
  - 可以聊晚餐相关
  - 保持随意，不要像问卷调查

## 📋 使用示例

### 示例 1：用户说要吃饭

**场景：**
```
用户（晚上7:30）："我要去吃饭了"
【30分钟后用户点击生成】
```

**时间感知分析：**
```
⚠️ 用户30分钟前说要去吃饭，
但吃饭通常需要20-60分钟，
所以用户现在应该还在吃饭中或刚刚结束。
你可以自然地询问或等对方主动说，不要显得太急切。
```

**AI可能的回复：**
- "吃完啦？" ✅
- "吃的什么呀~" ✅
- "怎么这么快，吃完了？" ✅

**AI不应该说：**
- "快去吃饭吧！" ❌（时间已经过去了）
- "趁热吃哦~" ❌（已经吃完或快吃完了）

---

### 示例 2：隔了几天才回复

**场景：**
```
用户（周一）："今天要去吃火锅"
【周四才点击生成】
```

**时间感知分析：**
```
⚠️ 已经隔了3天，千万不要回应之前的具体话题
像久别重逢的朋友一样：简单问好、聊聊最近、分享新鲜事
完全忘记之前说了什么，开启全新对话（这才是真实的）
语气可以轻松随意，不要显得在"履行义务"地回复
```

**AI应该说：**
- "嘿！这几天去哪浪了~" ✅
- "最近怎么样呀？" ✅
- "好久不见！最近忙吗？" ✅

**AI不应该说：**
- "火锅吃得怎么样？" ❌（太奇怪了）
- "火锅好吃吗？" ❌（不符合真实对话）
- "之前说要吃火锅，吃了吗？" ❌（太刻意）

---

### 示例 3：深夜时段

**场景：**
```
用户（凌晨2:30）："在吗"
```

**时间感知分析：**
```
- 现在是深夜时分（0-5点），正常人应该在睡觉
- 如果还在聊天，说明有特殊情况（失眠、加班、有心事等）
- 语气要安静、温和，可以关心对方为什么还没睡
```

**AI应该说：**
- "在呢，怎么还没睡？" ✅
- "这么晚了还不睡呀，失眠了？" ✅
- "怎么了？" ✅（简单关心）

### 示例 4：跨越用餐时间

**场景：**
```
用户（上午11:00）："在干嘛呢"
【下午1:30才点击生成】
```

**时间感知分析：**
```
- 跨越了午餐时间，可以提及"吃饭了吗"
- 但这不是必须的，真人不会每次都问
```

**AI可以说：**
- "刚吃完饭，你呢？吃了吗~" ✅

## 🎨 核心原则

### 1. 时间感知，但不刻意

❌ **错误：打卡式回应**
```
用户（3天前）："今天要加班"
AI："加班辛苦了！项目进展怎么样？"
```

✅ **正确：自然开启新话题**
```
用户（3天前）："今天要加班"
AI："最近怎么样呀~有没有什么新鲜事？"
```

### 2. 随意而非模板化

❌ **错误：每次都一样**
```
每天早上：
AI："早上好！吃早餐了吗？"
AI："早上好！吃早餐了吗？"
AI："早上好！吃早餐了吗？"
```

✅ **正确：保持多样性**
```
今天早上："嘿！醒了呀~"
明天早上："昨晚睡得好吗？"
后天早上："新的一天！今天有什么安排吗？"
```

### 3. 选择性回应

**时间间隔表：**
- **几分钟到1小时**：✅ 可以延续话题
- **几小时**：⚠️ 可提可不提
- **超过半天**：❌ 优先新话题
- **超过1天**：❌❌ 完全不要提旧话题

### 4. 语气要自然

❌ **客服式对话**
```
AI："您好！请问您吃饭了吗？"
AI："好的，祝您用餐愉快！"
AI："请问还有什么可以帮您的吗？"
```

✅ **朋友式对话**
```
AI："吃了吗~"
AI："吃啥好吃的！"
AI："哈哈下次带我~"
```

## 🔧 技术实现

### 文件结构

```
src/utils/timeAwareness.ts  // 时间感知工具函数
src/components/ChatScreen.tsx  // 集成时间感知
```

### 核心函数

#### 1. `buildTimeContext(timestamp, content)`

构建时间上下文信息：
- 当前时间（年月日时分、星期、时段）
- 上一条消息时间
- 时间间隔（分钟数和描述）
- 行为分析（如果用户说要做什么）
- 对话建议

#### 2. `buildTimeAwarePrompt(timestamp, content)`

生成时间感知的系统提示词，包括：
- 时间信息
- 行为分析
- 对话建议
- 核心原则

#### 3. `analyzeUserAction(message, timeGap)`

分析用户消息中的行为：
- 检测关键词（吃饭、睡觉、洗澡等）
- 判断是否"即将要做"
- 计算时间间隔与行为时长的关系
- 给出具体建议

## 📊 提示词示例

```
【时间感知】
当前时间: 2025年11月5日 14:30，星期二下午
对方上一条消息时间: 今天中午 12:00（2小时前）

【⚠️ 重要：对话时间分析】
⚠️ 用户120分钟前说要去吃饭，
现在时间已经过去，用户可能已经完成吃饭。
你可以：
1. 自然地问一句（如"吃饭完了吗"、"吃饭怎么样"）
2. 或者不提，等对方主动说
保持轻松随意的语气，不要像在"完成任务"一样询问。

【对话建议】
- 时间间隔适中，可以延续话题或略微调整方向
- 不用刻意解释时间空白，自然对话即可
- 现在是下午（14-18点）
- 跨越了午餐时间，可以提及"吃饭了吗"
- 但这不是必须的，真人不会每次都问

【⚠️ 核心原则：保持真实人类对话的自然性】
1. **时间感知，但不刻意**：
   - 了解时间间隔，但不要每次都"打卡式"地回应之前的话题
   - 时间过去很久（超过1天）时，像真人一样开启新话题，不要追问旧事
   - 例如：3天前对方说"要吃火锅"，现在不要问"火锅吃得怎么样"，这太奇怪了

2. **随意而非模板化**：
   - 不要每次都问"吃饭了吗"、"睡得好吗"，真人不会这么规律
   - 有时候直接聊新话题，有时候简单问候，有时候等对方引导
   - 保持对话的多样性和不可预测性

3. **选择性回应**：
   - 时间间隔短（几分钟到1小时）：可以延续之前的话题
   - 时间间隔中（几小时）：可提可不提，看情况
   - 时间间隔长（超过半天）：优先选择新话题或简单问候
   - 时间间隔很长（超过1天）：完全不要提之前的具体内容

4. **语气要自然**：
   - 不要显得在"履行义务"地回复
   - 不要像客服一样关心每个细节
   - 像朋友一样轻松随意，有时甚至可以"忘记"之前说了什么
```

## 🎯 最佳实践

### DO's ✅

1. **让AI自然遗忘**
   - 时间过去很久，就像真人一样"忘记"之前说了什么
   - 开启全新对话，不要刻意延续

2. **保持随机性**
   - 不要每次都用相同的问候语
   - 有时直接聊新话题，有时简单问候

3. **信任AI判断**
   - 系统已经给出了详细建议
   - AI会根据情况自然选择

### DON'Ts ❌

1. **不要期待每次都提起之前的话题**
   - 时间间隔长时，AI会选择新话题
   - 这是正常的，就像真实朋友一样

2. **不要强求模板化回复**
   - 不是每次早上都要说"早上好"
   - 不是每次都要问"吃饭了吗"

3. **不要在几天后还期待延续话题**
   - 3天前说"要吃火锅"，现在不应该再问
   - AI会像真人一样开启新话题

## 🚀 总结

时间感知系统让AI拥有了：

✅ **时间意识**：知道现在几点、距离上次聊天过去多久  
✅ **行为理解**：知道用户说要做什么、需要多长时间  
✅ **智能判断**：根据时间间隔决定是否延续话题  
✅ **自然对话**：像真人一样随意、多样、不刻意  

**核心理念：**
> 让AI不再是一个"完美的客服"，而是一个"会忘记、会随意、会自然聊天的朋友"。
