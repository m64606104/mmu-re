## 语音转文字配置步骤

### 1. 打开设置页面
在应用主屏幕点击"设置"图标

### 2. 配置语音转文字选项

找到"语音转文字"部分，需要填写以下信息：

- **启用语音识别**：打开开关
- **API 地址**：填写智谱清言API地址
  - 示例：`https://open.bigmodel.cn/api/paas/v4`
  - 注意：不需要包含 `/v1/chat/completions`，系统会自动添加
- **API Key**：填写您的智谱清言API密钥
- **识别模型**：选择语音识别模型
  - 默认：`glm-4-flash`
  - 可选：`glm-4-air`、`glm-4-plus` 等支持语音的模型

### 3. 保存配置
点击"保存配置"按钮

## 使用语音功能

1. 在聊天界面点击底部的麦克风图标
2. 允许浏览器访问您的麦克风（首次使用时）
3. 开始说话，完成后点击"停止录音"
4. 系统会自动识别您的语音并转换为文字
5. 确认或修改识别结果后点击"发送"

## 常见问题

### Q: 语音识别失败怎么办？
A: 
- 检查API地址和API Key是否正确
- 确保网络连接正常
- 查看浏览器控制台（F12）中的错误信息
- 如果自动识别失败，可以手动输入语音内容

### Q: 为什么没有识别结果？
A: 
- 确保录音时长至少1秒
- 检查麦克风权限是否已授权
- 确认使用的模型支持语音识别功能
- 检查录音环境是否过于嘈杂

### Q: 如何查看调试信息？
A: 
打开浏览器控制台（按F12），在Console标签页可以看到：
- 🎤 音频信息（大小、类型）
- 📡 API调用详情
- ✅ 识别成功信息
- ❌ 错误详细信息

## 调试技巧

如果遇到问题，请按以下步骤排查：

1. **检查配置**
   ```
   打开控制台，查找 "API配置:" 日志
   确认 url、model、hasKey 都正确
   ```

2. **检查音频**
   ```
   查找 "音频信息:" 日志
   确认 size > 0 且 type 为 audio/webm
   ```

3. **检查API响应**
   ```
   查找 "API响应状态:" 和 "API响应数据:"
   如果状态不是200，会显示详细错误信息
   ```
