# ❌ API错误速查手册

## 🔥 最常见错误

### ⏱️ 429 - 请求过于频繁

**你看到了什么：**
```
Failed to load resource: the server responded with a status of 429 (Too Many Requests)
或者
⏱️ 请求过于频繁
您在短时间内发送了太多请求，触发了API的速率限制
```

**这是什么意思：**
你的API请求太频繁了，像图片中显示的那样，短时间内发送了太多请求。就像排队买奶茶，你一个人占了好几个位置，店家就会让你等一等。

**立即解决（3步走）：**
1. **等待1-3分钟** - 这是最简单有效的方法
2. **刷新页面** - 清除当前排队的请求
3. **重新发送消息** - 问题应该就解决了

**如果频繁出现（长期方案）：**

✅ **方案1：升级API套餐**
```
大多数API服务商都有不同级别的配额
免费版 → 付费版 = 更高的请求限制
```

✅ **方案2：开启"自定义上下文数量"**
```
位置：角色设置 → 自定义上下文数量 → 开启
建议值：20条
效果：减少每次请求的数据量，降低被限流概率
```

✅ **方案3：关闭"完整记忆系统"**
```
位置：角色设置 → 完整记忆系统 → 关闭
效果：只在需要时调用记忆，减少请求复杂度
```

✅ **方案4：减少AI角色数量**
```
如果你创建了很多AI角色，朋友圈自动生成会频繁调用API
建议保持2-5个AI角色
```

### 🔑 401 - API密钥无效

**错误信息：**
```
🔑 API密钥无效
您的API密钥不正确或已过期
```

**可能原因：**
- API密钥输入错误（多/少了字符）
- API密钥已过期
- API密钥被撤销

**解决方法：**
```
1. 进入"设置 → API配置"
2. 检查API密钥是否完整
3. 去API服务商网站重新复制密钥
4. 粘贴到应用中并保存
5. 刷新页面重试
```

---

### 🔍 404 - API地址错误

**错误信息：**
```
🔍 API地址错误
无法找到指定的API服务
```

**常见错误：**
```
❌ https://api.openai.com/v1/chat/completion  (少了s)
✅ https://api.openai.com/v1/chat/completions

❌ http://api.openai.com/v1/chat/completions  (应该用https)
✅ https://api.openai.com/v1/chat/completions

❌ https://api.openai.com/chat/completions     (缺少/v1)
✅ https://api.openai.com/v1/chat/completions
```

**解决方法：**
```
1. 检查API地址拼写
2. 确认使用https://而不是http://
3. 确认地址包含完整路径（如/v1/chat/completions）
4. 参考API服务商文档确认正确地址
```

### 🔧 503 - API服务暂时不可用

**错误信息：**
```
🔧 API服务暂时不可用
API服务正在维护或负载过高
```

**这意味着：**
- API服务商正在维护
- 服务器负载过高
- 临时性故障

**解决方法：**
```
1. 等待5-10分钟后重试
2. 检查API服务商的状态页面
   OpenAI: https://status.openai.com
   Anthropic: https://status.anthropic.com
3. 如果长时间无法访问，联系服务商客服
```

## 📚 其他错误

### 💥 500 - API服务器错误

**不是你的问题！** 这是API服务商的服务器出了问题。

**解决：** 等待5-10分钟后重试，或联系API服务商。

---

### 📡 网络连接失败

**错误信息：**
```
📡 网络连接失败
无法连接到API服务器
```

**检查清单：**
```
✅ Wi-Fi是否连接正常？
✅ 能否访问其他网站？
✅ API地址是否正确？
✅ 是否需要使用VPN？
✅ 防火墙是否阻止了连接？
```

---

### ⏳ 请求超时

**错误信息：**
```
⏳ 请求超时
连接API服务器超时
```

**可能原因：**
- 网络速度慢
- 请求内容太大（消息太多）
- API服务器响应慢

**解决方法：**
```
1. 检查网络连接
2. 开启"自定义上下文数量"限制为10-20条
3. 稍后重试
```

## 💡 预防措施

### 避免429错误

```
1. 开启"自定义上下文数量"（推荐20条）
   - 减少每次请求的token数量
   - 降低被限流的概率

2. 关闭"完整记忆系统"
   - 按需调取记忆，而不是每次都加载
   - 显著减少API调用复杂度

3. 控制AI角色数量
   - 建议2-5个AI角色
   - 太多角色会导致朋友圈自动生成频繁调用API

4. 避免连续快速发送消息
   - 等AI回复完再发下一条
   - 不要短时间内发送大量消息
```

### 优化API配置

```
✅ 使用稳定的API服务商
✅ 选择合适的套餐（有足够配额）
✅ 定期检查API密钥有效性
✅ 确保API地址正确无误
```

## 🆘 仍然无法解决？

### 检查顺序

```
第1步：确认API配置
     → 设置 → API配置 → 检查地址和密钥

第2步：查看浏览器控制台
     → F12 → Console → 查看详细错误信息

第3步：测试API连接
     → 发送一条简单消息测试

第4步：尝试性能优化
     → 开启"自定义上下文"
     → 关闭"完整记忆系统"

第5步：等待后重试
     → 等待5-10分钟
     → 刷新页面后重试
```

### 获取帮助

```
📧 联系开发者
🔍 查看详细文档：
   - AI_MOMENTS_GUIDE.md（朋友圈功能）
   - PERFORMANCE_CONTROL.md（性能配置）
📊 提供错误信息：
   - 浏览器控制台截图
   - 错误代码和信息
   - API配置（隐藏密钥）
```

## 📊 错误严重程度

### 🟢 轻微（可自行解决）
```
401 - API密钥错误
404 - API地址错误
```

### 🟡 中等（需要等待或配置）
```
429 - 请求过于频繁
504 - 请求超时
```

### 🔴 严重（服务商问题）
```
500 - 服务器内部错误
502 - 网关错误
503 - 服务不可用
```
