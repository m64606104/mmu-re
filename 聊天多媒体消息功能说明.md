# 聊天多媒体消息功能说明

## ✅ 功能概述

本次更新实现了完整的多媒体消息功能，包括：
1. **用户发送表情包**（文字描述）
2. **AI发送图片、视频、语音、表情包**（文字描述）
3. **AI识别和理解用户的表情包**
4. **丰富的多媒体消息显示组件**

## 🎯 核心特性

### 1. 用户端功能

#### 📷 用户可发送的内容
- ✅ **图片**：上传真实图片文件
- ✅ **视频**：上传真实视频文件（带文字描述）
- ✅ **语音**：录制真实语音（带语音转文字）
- ✅ **表情包**：输入表情包的文字描述

#### 😊 表情包发送流程
1. 点击工具栏中的笑脸图标（Smile）
2. 在弹窗中输入表情包的文字描述
   - 例如："一只猫咪害羞捂脸"
   - 例如："小狗狗笑得很开心的样子"
3. 点击"发送"
4. 消息显示为表情包格式，带有表情图标标识

### 2. AI端功能

#### 🤖 AI可发送的内容
AI现在可以主动发送多种类型的消息：
- **图片**：半透明白色方块 + 图片描述
- **视频**：半透明白色长方形 + 视频描述
- **语音**：绿色语音气泡条 + 语音文字
- **表情包**：表情图标 + 表情描述

#### 📝 AI发送格式
AI使用特定的标记格式来发送多媒体消息：

```
[图片:夕阳西下的海边，天空渐变成橙红色]
[视频:小猫咪在草地上打滚玩耍的视频]
[语音:喜欢你呀，3秒]
[表情包:一只猫咪害羞捂脸]
```

## 💻 技术实现

### 类型扩展

#### Message类型新增字段
```typescript
interface Message {
  // ... 原有字段
  mediaType?: 'image' | 'video' | 'voice' | 'sticker'; // 新增sticker
  voiceDuration?: number; // 语音时长（秒）
  isVoicePlayed?: boolean; // 语音是否已播放
}
```

### AI提示词规则

#### 普通对话中的多媒体规则
```
【发送多媒体消息规则】：
你可以发送多种类型的消息，使用以下格式：
- 发送图片：[图片:详细的图片内容描述，10-50字，要生动具体]
- 发送视频：[视频:详细的视频内容描述，10-50字]
- 发送语音：[语音:语音内容的文字，时长X秒]
- 发送表情包：[表情包:表情包的详细描述]

使用场景：
- 想分享美景、照片时发图片
- 想分享有趣的视频时发视频
- 想发语音聊天时发语音（控制在3-10秒）
- 想表达情绪、开玩笑时发表情包

可以在文字消息中添加媒体，也可以单独发送媒体。
```

#### 表情包识别规则
```
【表情包理解规则】：
- 用户发送了表情包，根据描述的内容理解用户的情绪和意图
- 像朋友间日常聊天一样对表情包做出自然反应
- 可以回复文字、也可以回复表情包
- 根据表情包内容判断是否要发送图片/视频/语音/表情包回复
```

## 🎯 设计理念

### 为什么用文字描述？

1. **简化实现**：不需要真实的图片、视频生成
2. **成本控制**：避免调用图像生成API
3. **想象空间**：文字描述给用户留下想象空间
4. **模拟真实**：半透明占位符模拟真实媒体的存在感

### 为什么区分用户和AI？

1. **用户**：可以上传真实文件（图片、视频、语音）
2. **AI**：只能发送文字描述的"虚拟"媒体
3. **统一体验**：两者在聊天中都有媒体的视觉呈现
4. **清晰区分**：用户看得出哪些是真实的，哪些是AI生成的描述

## 🐛 故障排除

### AI不发送多媒体消息？
- 检查提示词是否正确设置
- 尝试在对话中明确提示："发个图片看看"
- 确认API配置正常

### 表情包显示异常？
- 检查消息类型是否为'sticker'
- 确认mediaDescription字段存在
- 查看控制台是否有错误

### 语音点击无反应？
- 检查viewingVoice状态管理
- 确认mediaDescription存在
- 查看事件绑定是否正确
